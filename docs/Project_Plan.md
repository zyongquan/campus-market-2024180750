# 校园轻集市 —— 项目规划

> 基于需求分析和实训计划制定

---

## 一、项目概述

**项目名称**: 校园轻集市（Campus Light Market）

**项目定位**: 面向高校学生的校园二手物品交易平台前端应用

**技术栈**: Vue 3 + TypeScript + Pinia + Vue Router + Vite

**开发周期**: 7 天

## 五、开发重点（个人判断）

### 1. 组件化设计（⭐⭐⭐⭐⭐）
- 将商品卡片、导航栏、搜索框等抽取为独立可复用组件
- 遵循"单一职责"原则，每个组件只做一件事

### 2. 状态管理（⭐⭐⭐⭐⭐）
- 使用 Pinia 管理用户登录状态、商品数据
- 合理划分 Store（userStore、goodsStore 等）

### 3. API 封装（⭐⭐⭐⭐）
- 统一请求拦截器（Base URL、Token、错误处理）
- 按模块组织 API 文件

### 4. 路由设计（⭐⭐⭐⭐）
- 合理规划路由结构，支持路由传参
- 添加路由守卫保护需要登录的页面

### 5. TypeScript 类型安全（⭐⭐⭐）
- 定义清晰的接口类型（Goods、User 等）
- 利用 TypeScript 减少运行时错误

## 六、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 后端 API 不可用 | 前端无法获取真实数据 | 使用 Mock 数据先行开发 |
| 组件拆分不合理 | 后期重构成本高 | 开始前花时间规划组件树 |
| 状态管理混乱 | 数据流难以追踪 | 严格控制 Store 职责边界 |
| 时间不足 | 功能无法全部完成 | 优先保证 P0 核心功能 |
